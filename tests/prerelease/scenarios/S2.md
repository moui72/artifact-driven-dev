# S2 — brownfield reverse-engineer init

Axes: acquisition = `install.sh` direct; state = brownfield (real,
never-ArDD codebase); mode = solo.

## Purpose

Run `/ardd-init`'s existing-codebase path cold on a real, lived-in app
and judge whether the reverse-engineered artifacts describe the code
that's actually there — accuracy, not just schema validity.

## Setup

- Clone a real non-ArDD local repo (default: `~/dev/daily-huddle`; the
  dispatcher may substitute) from its FILESYSTEM PATH into
  `$SCRATCH/s2/app`, then `git remote remove origin` immediately.
- Install ArDD from the local source checkout:
  `<ardd-source>/install.sh $SCRATCH/s2/app`. Record install output.

## Steps

1. Run `/ardd-init` in the clone. It should detect existing code and take
   the reverse-engineer path without an interview restart.
2. Spot-check accuracy programmatically where possible: pick 5 concrete
   claims the generated artifacts make (a dependency, a data shape, an
   entry point, a directory role, a listed feature) and verify each
   against the actual files with grep/read. Every miss is a finding with
   the claim, the file, and the truth.
3. Judge artifact `status` values: reverse-engineered artifacts marked
   `stable` without evidence is a finding.
4. Run `/ardd-lint`, `/ardd-status`, and `/ardd-defects`; on a
   just-initialized project DEFECTS should be empty or explain itself.

## Scripted answers

- `/ardd-init`: existing-codebase path; accept its detection; artifact
  set: whatever it proposes (record the proposal and judge it);
  `workflow_mode: solo`; `next_step_prompt: false`.
- Any next-step prompt: decline.

## Checklist

- [ ] existing-code detection fired (no greenfield interview)
- [ ] 5/5 spot-checked claims accurate (list each)
- [ ] statuses honest (draft where unverified)
- [ ] lint-clean on first try
- [ ] defects run on fresh init behaves sensibly
