# S3 — consumer upgrade + newest features

Axes: acquisition = `/ardd-update` on an existing install several
releases behind; state = established ArDD consumer; mode = as-found.

## Purpose

Exercise the upgrade path (migrations, version recording, channel
handling) on a real consumer, then stress-test whichever flags/features
shipped MOST RECENTLY — the dispatcher names them in the run prompt
(2026-07-15 example: `/ardd-update --reconfigure`, shipped hours before).

## Setup

- Clone a real ArDD consumer (default: `~/dev/atelier`) from its
  FILESYSTEM PATH into `$SCRATCH/s3/app`; `git remote remove origin`.
- Do NOT reinstall first — the point is upgrading the stale install the
  clone carries. Note the recorded `Source-Ref`/`Channel` before.

## Steps

1. Run `/ardd-update`. It resolves the recorded source; if the recorded
   `Source-Path` is unusable in the sandbox, point it at the local ArDD
   source checkout and note that this makes it dev-mode (expect the
   dev-mode warn-and-ask; answer per Scripted answers).
2. Verify after: new version recorded, `.ardd-applied` gained exactly the
   migrations between the two versions, skills refreshed, no tracked-file
   damage in the consumer's own content.
3. Stress the dispatcher-named recent feature(s) (e.g.
   `/ardd-update --reconfigure`: re-ask `workflow_mode` and
   `next_step_prompt`, flip each, verify `ardd-state.sh stamp` wrote them
   and lint accepts).
4. Run `/ardd-status` and `/ardd-lint` post-upgrade on the consumer's
   real `.project/` — pre-existing consumer quirks are context, new
   breakage from the upgrade is a bug.

## Scripted answers

- dev-mode warning: proceed.
- update field backfill prompts (e.g. missing `next_step_prompt`): accept
  default.
- `--reconfigure`: flip both fields, then flip back.
- Any next-step prompt: decline.

## Checklist

- [ ] update completed; version/channel recorded correctly
- [ ] migrations applied exactly once, in order
- [ ] recent-feature stress items behaved as documented
- [ ] no consumer-content damage
- [ ] post-upgrade lint/status coherent
