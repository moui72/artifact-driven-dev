## Purpose

Exercise the upgrade path (migrations, version recording, channel
handling) on a real consumer, then stress-test whichever flags/features
shipped MOST RECENTLY â€” the dispatcher names them in the run prompt
(2026-07-15 example: `/ardd-update --reconfigure`, shipped hours before).

## Setup

- Clone a real ArDD consumer (default: `~/dev/atelier`) from its
  FILESYSTEM PATH into `$SCRATCH/s3/app`; `git remote remove origin`.
- Do NOT reinstall first â€” the point is upgrading the stale install the
  clone carries. Note the recorded `Source-Ref`/`Channel` before.

## Steps

1. Run `/ardd-update`. It resolves the recorded source; if the recorded
   `Source-Path` is unusable in the sandbox, point it at the local ArDD
   source checkout and note that this makes it dev-mode (expect the
   dev-mode warn-and-ask; answer per Scripted answers).
2. Verify after: new version recorded, `.ardd-applied` gained exactly the
   migrations between the two versions, skills refreshed, no tracked-file
   damage in the consumer's own content.
3. Stress the dispatcher-named recent feature(s) (e.g.
   `/ardd-update --reconfigure`: re-ask `workflow_mode` and
   `next_step_prompt`, flip each, verify `ardd-state.sh stamp` wrote them
   and lint accepts).
4. Run `/ardd-status` and `/ardd-lint` post-upgrade on the consumer's
   real `.project/` â€” pre-existing consumer quirks are context, new
   breakage from the upgrade is a bug.
5. Channel-switch flow: on the same consumer, run `/ardd-update --beta`.
   Verify `.project/ardd-version.md`'s `Channel:` and `Source-Ref:` stay
   mutually consistent afterward (a beta `Channel:` paired with a
   prerelease `Source-Ref:`, never a stable-tag `Source-Ref:` left
   stranded under `Channel: beta`) â€” and confirm
   `lint-project.sh`'s prerelease-ref-under-stable check (from
   `channel-source-ref-consistency`) does NOT fire immediately after
   this switch, since the recorded state is self-consistent. Then run
   `.claude/skills/ardd-scripts/ardd-update-check.sh` and confirm
   `behind`/`up-to-date` is computed within the beta channel (comparing
   against the newest beta or stable, whichever is newer under the
   suffix-ordering rule â€” not silently falling back to stable-only
   comparison).
6. Run `/ardd-update --stable` on the same consumer. Verify `Channel:`
   flips back to `stable` and `Source-Ref:` records a strict `vX.Y.Z`
   tag (no beta suffix), and that `ardd-update-check.sh` now computes
   `behind` against the latest stable release only (prereleases
   invisible again). Confirm the lint check stays silent here too â€” the
   post-switch state is consistent.
7. Negative check: hand-edit `.project/ardd-version.md` to simulate a
   stale beta `Source-Ref:` left under `Channel: stable` (the actual bug
   class `channel-source-ref-consistency` was written to catch), run
   `lint-project.sh`, and confirm it DOES fire here â€” naming the file,
   both values, and the mismatch nature. Revert the hand-edit
   afterward.

## Scripted answers

- dev-mode warning: proceed.
- update field backfill prompts (e.g. missing `next_step_prompt`): accept
  default.
- `--reconfigure`: flip both fields, then flip back.
- Any next-step prompt: decline.
