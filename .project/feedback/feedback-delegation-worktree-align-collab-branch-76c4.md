# Feedback

## Bugs
- [ ] F001 `/ardd-implement`'s collaborative-mode delegation path has bad DevX: because `worktree-align.sh` defaults to aligning a fresh delegated worktree against the *local default branch* (`main`), a plan run whose plan/tasks files live on a feature branch (the normal collaborative-mode case, since nothing may be committed to local `main`) can't be delegated without first committing on the branch, running `fold-to-main.sh` (a local-ref fast-forward of `main` itself — no new commit, but it does make local `main` momentarily carry unreviewed feature work, which sits awkwardly against "nothing may be committed to the local default branch, ever"), pushing, and opening a draft PR — all *before* delegation can even start. This was hit live this session (`chore/feedback-status-readiness-review`, `workflow_mode: collaborative`) and confirmed by a design consult (fable-model agent, full transcript available on request) that read `worktree-align.sh`, `fold-to-main.sh`, and the relevant `/ardd-implement` SKILL.md sections directly.

  **Fix direction, confirmed against the actual scripts**: `worktree-align.sh` already accepts an optional `<ref>` argument (it only defaults to the local default branch when the argument is omitted — nothing about the script is main-specific). So in collaborative mode, the delegation preamble should pass the *current feature branch* as that ref (`worktree-align.sh <feature-branch>`) instead of relying on the default-argument behavior — worktrees share local branch refs, so this ff-merges the feature branch's own commits straight into the fresh worktree, with no `fold-to-main.sh` step and no local-`main`-touching at all. This is better than the alternative of having each delegated subagent `cp` the plan/tasks files in from the primary checkout's absolute path and commit them itself (also considered and rejected): ref-alignment transports *everything* the plan run touched (closing the general case of the artifact-gap class of bug — of which the already-planned `feedback-delegation-preflight-artifact-gap-44dc.md` fix only covers plan+tasks+features+feedback+artifacts explicitly, not anything else a future plan run might touch), and gives parallel worktrees shared commit ancestry for the same content rather than each minting its own divergent commit.

  **Scope/edge cases the implementing plan should address** (from the design consult): (1) `/ardd-implement` step 3's collaborative-mode branch of the delegation gate should skip the fold step entirely and instead pass `branch-info.sh`'s `current` (or `default`, if `on_default` is already true) as the align ref; (2) the "push + draft PR" offer becomes optional/visibility-only rather than a delegation prerequisite, since ref-alignment no longer depends on anything having reached `origin`; (3) on `aligned=false reason=diverged`, stop and surface verbatim — same refuse-don't-resolve discipline the fold path already has, not a new failure mode; (4) solo mode is explicitly **unaffected** — it should keep `worktree-align.sh`'s no-argument (default-branch) behavior and keep using `fold-to-main.sh` for its own on-a-branch eager-background case (decision record 0004), since solo's `merge_policy` eager-merge/reap flow assumes worktree branches merge into local default, which is still correct there; (5) a caveat worth stating in the SKILL prose once implemented: a worktree aligned to feature-branch A can't see uncommitted work still sitting only on unrelated feature-branch B — acceptable (the same-file claim check already serializes same-plan work, and a fan-out from one plan all aligns to the same branch), but should be documented, not silently assumed; (6) once a delegated branch's PR (which now carries the plan/tasks commits too) merges, the original plan-only draft PR becomes redundant and should be noted as closable/absorbable in the prose, mirroring what happened live this session (PR #13 vs. #14/#15 in the artifact-driven-dev repo itself).
