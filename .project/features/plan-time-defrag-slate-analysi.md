Plan-time 'defrag'/slate analysis: an advisory, recomputed-at-plan-time footprint analysis over open backlog items proposing session-optimized slates — bundles of items with overlapping code footprints (implement together, serially, in one plan) and pairwise-disjoint sets (separate tasks files for worktree fan-out).
Why: get maximum wallclock value from an implement session against a large backlog. Never stored in the register (computed/mechanical/ephemeral — distinct from the declared/semantic/durable epics-grouping-in-feature-regi); every footprint estimate grounded in actual greps of the codebase. Spec sources: two prototype /ardd-research runs, now both complete. First — sync-tab-scroll, 2026-07-15 (research-backlog-defrag-slate-analysis-2026-07-15-627c.md), N=1 backlog, established the method and five requirements (dependency ordering as a third axis beyond file overlap — overlap != parallelizable; grade footprint confidence and never place speculative items in parallel sets; read item status from the register directly, never STATUS.md counts; handle degenerate N=0/N=1 backlogs). Second — atelier, 2026-07-15 (research-atelier-backlog-defrag-slate-analysis-2026-07-15-b872.md, in that repo, uncommitted), N=12 backlog, the real cross-item validation the first prototype couldn't reach: all five requirements held up — the overlap-vs-dependency split did the most work (caught a same-seam-label item, project-scoped-personal-dictionary, that shares a *label* with the spellcheck bundle but not its actual file, `speller.ts` vs `personal-dictionary.ts`); STATUS.md happened to be accurate this run, reinforcing that the register-direct-read discipline should stay unconditional rather than conditional on staleness. New finding from N=12: hand-computing all pairwise overlap/dependency comparisons doesn't scale much past a dozen items — a codified slate mode will need an actual two-axis (overlap × dependency) graph structure and a cheaper pre-filter, not pairwise brute force.
Both prototypes complete; the method is validated at both N=1 and N=12. Next step: this feature is ready to move past backlog — /ardd-plan plan-time-defrag-slate-analysi can design the codified /ardd-plan slate-mode capability itself, informed by both research docs' findings (especially the graph-structure/pre-filter requirement the N=12 pass surfaced).
