A single curl-to-sh command creates a new project directory, installs ARDD into it via install.sh, and opens a primed first session that interviews the user and runs /ardd-bootstrap.
Why: greenfield onboarding currently spans four steps across two tools (clone or npx skills add, /ardd-setup, design conversation, /ardd-bootstrap); the cold-start case — nothing installed yet — has no single entry point.
