# Plan — principle-agnostic skills

## Goal

Make `/ardd-plan` naive to *which* principles a target constitution declares:
enforce whatever principles are actually present, and stop presuming a fixed
principle vocabulary (a "simplicity / Complexity Tracking" principle, a
"production-annotations" principle) that many target constitutions won't have.

## Scope

**In scope** — three prose edits to `skills/ardd-plan/SKILL.md`, the product
(the `.claude/skills/ardd-plan/` copy is a gitignored install artifact
regenerated by `install.sh`; it is *not* edited directly):

1. Step 6 ("Check constitution compliance") — currently flags patterns
   "per the simplicity principle" unconditionally.
2. Step 8 deliverable **Complexity Tracking** — currently "table of justified
   deviations from the simplicity principle (if a constitution is present)".
3. Step 8 deliverable **Production Annotation Summary** — currently listed
   unconditionally, presuming a production-annotations principle.

**Explicitly out of scope** (settled during planning):

- **`/ardd-critique`** — its `### Simplicity` heading (and the Failure
  modes / Robustness / … lenses) are *Claude's own review lenses* applied to
  artifacts regardless of the constitution, not a claim that the constitution
  defines a simplicity principle. Not the bug the feedback describes; left
  untouched (user decision).
- **The "constitution Principle II" meta-references** in `/ardd-render`
  (step 6), `/ardd-tasks` (step 60), and `/ardd-plan` (steps 4 & 9). These
  name *ARDD's own* constitution (determinism/mechanization) as maintainer
  commentary explaining why the skill shells out to a deterministic script.
  They are not target-constitution assumptions and must stay (feedback scope
  guard). This is the standing distinction between the two install targets
  in `CLAUDE.md` — a check/reference against *this* repo vs. a *target*
  project — applied to prose.

Note on the `[artifacts: constitution]` tag: it flagged the feedback as
constitution-*related*, but the deliverable is skill prose, not an edit to
this repo's own `.project/artifacts/constitution.md`. No artifact content
changes; no constitution version bump.

## Technical Approach

There is a proven in-repo precedent to mirror: **`/ardd-analyze` step 3**
already conditions correctly —

> Flag shortcuts that lack a production annotation entry (**if the
> constitution includes a production annotation principle**).

Apply the same "read the declared principles; act only on what's present"
shape to `/ardd-plan`'s three sites. A plan run should:

- read the principles `constitution.md` actually declares (not a fixed list);
- check the plan against those declared principles;
- produce a **Complexity Tracking** table only when a simplicity /
  complexity-justification principle is among them (else omit the section);
- produce a **Production Annotation Summary** only when a
  production-annotations principle is among them (else omit the section).

The result is symmetric with `/ardd-analyze` and correct for a target whose
constitution declares neither principle.

## Phase Breakdown

Single phase — a self-contained prose refactor of one file.

### Phase 1 — make ardd-plan principle-agnostic

- **T001** Rewrite `/ardd-plan` step 6 so it reads the constitution's
  *declared* principles and flags patterns that violate or need justification
  under any of them, rather than naming "the simplicity principle"
  unconditionally; the Complexity-Tracking flag becomes conditional on a
  simplicity/complexity principle actually being declared. `[artifacts: constitution]`
  (addresses feedback F001 — the step-6 site)
- **T002** Rewrite step 8's **Complexity Tracking** deliverable to be
  included only when the constitution declares a principle requiring
  complexity to be justified; omit the section entirely otherwise.
  `[artifacts: constitution]` (F001 — the step-8 Complexity site)
- **T003** Rewrite step 8's **Production Annotation Summary** deliverable to
  be included only when the constitution declares a production-annotations
  principle, mirroring `/ardd-analyze` step 3's condition; omit otherwise.
  `[artifacts: constitution]` (F001 — folded-in production-annotation site,
  user decision)

Demonstrable increment: a plan drafted against a constitution that declares
*neither* principle contains no Complexity Tracking and no Production
Annotation Summary section; one drafted against a constitution that declares
both contains both — with no other behavior changed.
